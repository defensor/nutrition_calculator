## 2024-03-14 - Modal Component Accessibility Pattern
**Learning:** The application's custom Modal component (`frontend/src/components/ui/Modal.jsx`) lacked fundamental accessibility features (`aria-modal`, `role="dialog"`, focus rings, `aria-labels` on icons) and a backdrop dismissal interaction. This pattern might be present in other custom popovers or dialogs in this specific app, indicating a general need for tighter accessibility standards when building custom UI components instead of using headless UI libraries.
**Action:** Always ensure custom dialogs in this design system include standard WAI-ARIA dialog attributes, visual focus states for keyboard users, and expected backdrop interactions. Consider recommending a transition to headless UI components (like Radix UI) for such primitives in the future.
